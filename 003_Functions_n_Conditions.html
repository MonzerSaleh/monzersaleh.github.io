Functions¶

def FunctionName([arg1, arg2, ... arg_n]):
    """docstring"""     # optional
    <statements>
    return arg          # optional

def -- keyword indicating the start of a function definition -- must be followed by the function signature Notes:

  1. The first line of the function body should be a docstring describing the function
  2. All variable assignments in a function are stored in a local symbol table
  3. Global variables cannot be assigned a value within a function (unless named in a global statement)
  4. Arguements are passed using "call by value" where value is always an object reference
  5. Arguements can be passed both by their name or by position. But I would recommend mixing the two to avoid confusion
  6. A function definition creates the function name in the local symbol table. The value of the function name will be recognized by the interpretor as a user defined type. Thus this type can be re-assigned a different name. This allows functions to be passed as arguements.
  7. Functions without a return statement return "None".

Variadic arguements¶

*args -- receives a non-keyworded variable length arguement list

**kwargs -- receives a keyworded dictionary containing all keyword args excluding formal parameters

These allow you to pass a variable number of arguements to a function. The names args and kwargs are conventional and not required but the astrisk is mandatory.

NB:

  1. Any basic parameters must come before arguements preceeded by * or **
  2. Both are optional however *args cannot come after **kwargs

Examples:

def test_var_args(f_arg, *argv):
    print "first normal arg:", f_arg
    for arg in argv:
        print "another arg through *argv :", arg

def greet_me(**kwargs):
    if kwargs is not None:
        for key, value in kwargs.iteritems():
            print "%s == %s" %(key,value)

test_var_args('yasoob','python','eggs','test')
greet_me(name="yasoob")

Handy Functions and constructs¶

in -- keyword similar to in in sql -- tests if a variable is in a sequence -> if n in ('a','b','c','d')

range(a) range(a, b[, c])
-- creates a sequence/list of numbers from a to b using steps of size c -- b & c are optional parameters. When left out the list begins from 0 to a-1 enumerate(iterable, start=0) -- returns an enumerate object. iterable must be a sequence, an iterator, or an object that supports iteration -- the next() method of the iterable returned by enumerate() is a tuple with a counter beginning at 0.
