Scopes & Namespaces¶

Define:

  1. A namespace is a mapping from names to objects, currently implemented in python as dictionaries.
  2. An attribute is basically any name following a dot, which can refer to both variables and functions.
  3. Scope is a textual region where a namespace is directly accessible. "Directly Accessible" simply means that an unqualified reference to a name will search for the name in the namespace.

At any point during the execution of a program there are at least 3 nested scopes

  1. The innermost scope, which is always searched first.
  2. The scope of any enclosing functions
  3. The next to last scope containing the current module's global names
  4. The outermost scope, searched last, is the namespace containing built in names

        return 'Hello'

Inheritance¶

class DerivedClassName(BaseClassName)
    <statement_1>
    .
    .
    .
    <statement_N>

The name BaseClassName must be defined in a scope containing the derived class definition. However, In place of a base class name, other arbitrary expression are also allowed. This can be helpful when deriving from a base class in another module

class DerivedClassName(modname.BaseClassName):

Execution of a derived class definition proceeds the same as for a base class. When the class object is constructed, the base class is remembered. This is used for resolving attribute references: if a requested attribute is not found in the class, the search proceeds to look in the base class. This rule is applied recursively if the base class itself is derived from some other class.

Derived classes may override methods of their base class. This can cause a base class to call a method in the derived class whenever it's been over-written. To explicitly call the baseclass method simply call BaseClassName.method(self,args).

Python has two functions to check for inheritance attributes isinstance() to check an instance's type against another issubclass() to check if a class inherits from another

Multiple inheritance follows a similar syntax¶

    <statement_N>

Name mangling & Private Variables¶

Private variables similar to other languages do not exist in python. However, there is a general convention that variables beginning with an underscore should be treated as a non-public part of an API. Any identifier beginning with two underscores, and possibly one trailing underscore, will be textually replace with the same name plus the prefix '_classname'. This is called name mangling and can be helpful in letting a subclass override base methods without breaking intraclass method calls.

In [11]:
class Mapping:
    def __init__(self, iterable):
        self.items_list = []
        self.__update(iterable)
        
    def update(self,iterable):
        for item in iterable:
            self.items_list.append(item)
            
    __update = update #a private copy of original update() method
    
class MappingSubClass(Mapping):
    
    def update(self,keys,values):
        # provides a new version without breaking the base class __init
        for item in zip(keys,values):
            self.items_list.append(items)
# zip() is a built-in function
